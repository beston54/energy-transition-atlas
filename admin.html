Don't have a token yet?

A token is like a one-time password for this computer. It lets the Atlas save your edits back to GitHub.

  1. Open github.com/settings/personal-access-tokens/new in a new tab (sign in to GitHub if asked).
  2. Token name: something you'll recognise, e.g. "Atlas Admin — my laptop".
  3. Expiration: 90 days is a good default. You'll need to repeat these steps to renew it.
  4. Repository access: choose Only select repositories and pick RenewablesGridInitiative/energy-transition-atlas.
  5. Scroll to Permissions → Repository permissions and set Contents to Read and write.
  6. Scroll to the bottom, click Generate token, then copy the token (starts with github_pat_).
  7. Paste it into the field on the left and click Sign In.

Lost your token? GitHub can't show it again — you'll need to generate a new one. That's fine, just repeat these steps.

If none of this looks familiar, ask whoever set up the Atlas for help — don't guess.
